# A1-5　快速排序法（Quick Sort）

由 C.A.Hoare 所發明的快速排序法是目前大家所公認的最佳排序法。雖然它的排序方法和泡沫排序法一樣是使用交換的方法來排序，但透過分割的技巧，快速排序法名符其實的遠遠超過泡沫排序法的執行效率。

---

## 一、核心概念

快速排序法的核心是**分割（Partition）**：每次從資料中選出一個**基準元素（Pivot）**，將資料分成兩半：

- 一半是**比基準值小或相等**的元素
- 另一半是**比基準值大**的元素

接著對每一半遞迴地重複相同操作，直到每個子集合無法再分割為止，排序就完成了。

---

## 二、基準元素的選法

最佳的做法是找出中間值，但對大部分資料而言不易實作，常用兩種替代方式：

1. 取欲分割陣列的**第一個元素**（本節採用此方法）
2. 取欲分割陣列**最中間**的那個元素

---

## 三、分割步驟示範

以下面這個字元陣列（索引從 0 開始）為例：

<img src="/img/fig_a1-5-1.svg" alt="初始陣列" width="90%">

選 `d`（索引 0）為基準元素，分割方法如下：

> 由左至右掃描，找到第一個**大於** `d` 的元素（左指標）；
> 同時由右至左掃描，找到第一個**小於** `d` 的元素（右指標）；
> 將兩元素互換，繼續掃描，直到左指標 > 右指標為止。

---

### 🔷 第一步：左右指標找目標

<img src="/img/fig_a1-5-2.svg" alt="左右指標找到目標" width="90%">

- **左指標**由左向右掃描，在索引 1 找到 `e`（`e > d`）
- **右指標**由右向左掃描，在索引 5 找到 `b`（`b < d`）

---

### 🔷 第二步：交換，繼續掃描直到指標交叉

<img src="/img/fig_a1-5-3.svg" alt="交換後指標交叉" width="90%">

① 交換 `e`（索引 1）↔ `b`（索引 5），陣列變為 `[d, b, a, c, f, e, h, g]`。

② 繼續掃描：左指標推進到索引 4（`f > d`），右指標退回到索引 3（`c < d`）。
此時**左指標 > 右指標**，停止掃描。

---

### 🔷 第三步：基準元素就定位

將基準元素 `d`（索引 0）與右指標所指的元素 `c`（索引 3）互換：

<img src="/img/fig_a1-5-4.svg" alt="基準元素就定位" width="90%">

**第一次分割完成！**

- 索引 3 的 `d` 已永久就定位（藍綠色）
- **左分割區** `[c, b, a]`（索引 0–2）：元素均 < `d`
- **右分割區** `[f, e, h, g]`（索引 4–7）：元素均 > `d`

## 五、C++ 程式碼

<CppRunner has-stdin>

```cpp:line-numbers
#include <iostream>
using namespace std;

// 分割函式：回傳基準元素最終所在索引
int partition(char arr[], int low, int high) {
    char pivot = arr[low];    // 選第一個元素為基準
    int L = low + 1;          // 左指標從 low+1 開始
    int R = high;             // 右指標從 high 開始

    while (true) {
        while (L <= R && arr[L] <= pivot) L++;  // 找到 > pivot
        while (L <= R && arr[R] >= pivot) R--;  // 找到 < pivot
        if (L > R) break;
        swap(arr[L], arr[R]);
    }
    swap(arr[low], arr[R]);   // 基準元素就定位
    return R;
}

// 快速排序主函式（遞迴）
void quickSort(char arr[], int low, int high) {
    if (low >= high) return;
    int mid = partition(arr, low, high);
    quickSort(arr, low, mid - 1);   // 左半部遞迴
    quickSort(arr, mid + 1, high);  // 右半部遞迴
}

int main() {
    int n;
    cin >> n;
    char arr[n];
    for (int i = 0; i < n; i++)
        cin >> arr[i];

    quickSort(arr, 0, n - 1);

    for (int i = 0; i < n; i++)
        cout << arr[i] << " ";
    cout << endl;
    return 0;
}
```

## 六、時間複雜度分析

| 情況 | 時間複雜度 | 說明 |
|:---|:---:|:---|
| 最佳（每次均分） | O(n log n) | Pivot 每次都接近中位數 |
| 平均 | O(n log n) | 隨機資料下的期望值 |
| 最壞（已排序資料） | O(n²) | Pivot 每次都是最大或最小值 |
| 空間複雜度 | O(log n) | 遞迴呼叫堆疊深度 |

::: tip 為什麼快速排序通常最快？
兩個演算法都是 O(n log n)，但實際執行時間大約是：

- 快速排序：**1 × n log n**
- 合併排序：**3 × n log n**（因為需要額外複製陣列）

這個前面的 1 和 3 就叫「**常數係數**」。O 符號會把它忽略掉，所以看起來一樣快，但實際上差很多。加上快速排序是原地排序，不需要額外記憶體，實際效能通常最優。
:::

::: warning 最壞情況的防範
當輸入資料已接近排序完成時，若總是選第一個元素為 Pivot，效能會退化為 O(n²)。常見解法是改選**隨機元素**或**三數取中（median-of-three）**作為基準元素。

**三數取中**的做法：取陣列的第一個、中間、最後一個元素，選這三個值的中間值作為 Pivot。

例如陣列 `[8, 3, 1, 5, 9, 2, 6, 4]`：

- 第一個 = **8**，中間 = **9**，最後 = **4**
- 三個值排序後：4 < **8** < 9，中間值是 8
- 選 **8** 作為基準元素

這樣能避免 Pivot 每次都選到極端值，讓左右兩區大小更平均，效能更穩定。
:::
