# a1-4　累堆排序法（Heap Sort）

累堆排序法是一種以**二元樹**為基礎的排序演算法，核心概念是利用「累堆（Heap）」的特性，每次快速取得目前最大（或最小）的元素。


## 一、什麼是累堆（Heap）？

累堆是一棵**完全二元樹**，必須滿足下列條件：

> **每個父節點的資料，都大於或等於其左、右子節點的資料。**

符合此條件的累堆稱為 **Max-Heap（最大累堆）**，根節點永遠是整棵樹的最大值。

下圖示範一棵有 9 個節點的合法 Max-Heap，以及它對應的陣列表示：

<img src="/img/fig_a1-4-1.svg" alt="Max-Heap 範例" width="90%">

::: tip 為什麼根節點一定最大？
因為任何節點都比它的子節點大，沿著任何一條從根到葉的路徑，值一定是遞減的，所以根節點必然是整棵樹的最大值。
:::



## 二、二元樹的陣列表示法

累堆通常以**一維陣列**儲存，索引從 **1** 開始（不使用 `a[0]`），節點位置對應關係如下：

| 節點 | 陣列索引 |
|:---:|:---:|
| 根節點 | `1` |
| 節點 `n` 的左子節點 | `2 × n` |
| 節點 `n` 的右子節點 | `2 × n + 1` |
| 節點 `n` 的父節點 | `⌊n / 2⌋` |

以下圖為例，9 個元素的初始二元樹（尚未建成累堆）：

## 三、累堆排序的兩個階段

累堆排序分成兩大步驟：

1. **建立累堆** ── 將任意二元樹調整成 Max-Heap
2. **取出排序** ── 反覆把根（最大值）移走，重建累堆，直到全部取完

---

## 四、第一階段：建立累堆

### 4.1 建堆策略

以含有 `n` 個元素的陣列為例，葉節點不需要調整（它們沒有子節點），因此**從索引 `⌊n/2⌋` 往前**，逐一調整每個非葉節點，使其滿足累堆性質。

調整一個節點的規則（稱為 **Heapify** 下沉操作）：

1. 找出左子節點、右子節點中**較大的那個**。
2. 若該子節點**大於**目前節點，則**交換**兩者資料。
3. 交換後，對被換下去的子節點**繼續往下**執行相同操作（因為交換可能破壞下方的累堆性質）。
4. 若目前節點已大於等於兩個子節點，**停止**。

### 4.2 逐步示範

以陣列 `a[] = [_, 5, 6, 4, 8, 2, 3, 7, 1, 9]`（`_` 表示不使用的 `a[0]`）為例，`n = 9`，從索引 `⌊9/2⌋ = 4` 開始。

---

#### 🔶 步驟一：處理索引 4（值 = 8）

索引 4 的左子節點是索引 8（值=1），右子節點是索引 9（值=9）。  
右子節點 9 > 8，**進行交換**。

<img src="/img/fig_a1-4-3.svg" alt="建堆步驟1：比較索引4" width="90%">

交換後：索引 4 = **9**，索引 9 = **8**。索引 9 沒有子節點，停止。

---

#### 🔶 步驟二：處理索引 3（值 = 4）

索引 3 的左子節點是索引 6（值=3），右子節點是索引 7（值=7）。  
右子節點 7 > 4，**進行交換**。

<img src="/img/fig_a1-4-4.svg" alt="建堆步驟2：索引4↔9交換完成，比較索引3" width="90%">

交換後：索引 3 = **7**，索引 7 = **4**。索引 7 沒有子節點，停止。

---

#### 🔶 步驟三：處理索引 2（值 = 6）

索引 2 的左子節點是索引 4（值=9），右子節點是索引 5（值=2）。  
左子節點 9 > 6，**交換**索引 2 與索引 4。

<img src="/img/fig_a1-4-5.svg" alt="建堆步驟3：比較索引2" width="90%">

交換後索引 2 = **9**，索引 4 = **6**。交換後索引 4（值=6）仍有子節點，繼續往下比較：左子索引 8（值=1），右子索引 9（值=8）。右子 8 > 6，**再次交換**。

<img src="/img/fig_a1-4-6.svg" alt="建堆步驟3串聯：索引4與子節點比較" width="90%">

交換後索引 4 = **8**，索引 9 = **6**。索引 9 沒有子節點，停止。

<img src="/img/fig_a1-4-7.svg" alt="步驟3串聯交換完成" width="90%">

---

#### 🔶 步驟四：處理索引 1（根節點，值 = 5）

目前根節點索引 1 的值為 5，左子索引 2（值=9），右子索引 3（值=7）。  
左子 9 > 5，**交換**索引 1 與索引 2 → 9 成為新根。

<img src="/img/fig_a1-4-8.svg" alt="建堆步驟4：根節點5↔9交換" width="90%">

交換後索引 2（值=5）繼續往下：左子索引 4（值=8），右子索引 5（值=2）。  
左子 8 > 5，**交換**索引 2 與索引 4。

<img src="/img/fig_a1-4-9.svg" alt="建堆步驟4：索引2(5)↔索引4(8)交換" width="90%">

交換後索引 4（值=5）繼續往下：左子索引 8（值=1），右子索引 9（值=6）。  
右子 6 > 5，**交換**索引 4 與索引 9。索引 9 無子節點，停止。

<img src="/img/fig_a1-4-10.svg" alt="Max-Heap 建立完成" width="90%">

**Max-Heap 建立完成！** 陣列變成 `[9, 8, 7, 6, 2, 3, 4, 1, 5]`，根節點 9 是整棵樹的最大值。原本位於根（索引 1）的值 5，經過四次串聯交換，最終沉降至索引 9（最後一個位置）。

---

## 五、第二階段：取出排序

Max-Heap 建立後，透過以下步驟將根節點（最大值）逐步移到陣列末尾，完成排序：

1. 將**根節點**（最大值）與**堆的最後一個元素**互換 → 最大值就定位於陣列末尾。
2. 將堆的有效範圍縮小一格（末尾元素已排好，不再參與調整）。
3. 對新的根節點執行 **Heapify（下沉操作）**，重建累堆性質。
4. 重複直到堆只剩 1 個元素。

### 5.1 第一回：取出最大值 9

從已建好的 Max-Heap `[9, 8, 7, 6, 2, 3, 4, 1, 5]` 開始：

**① 根節點 9 與末尾節點 5 互換** → 9 移至陣列最後，堆縮小為 8 個元素：

<img src="/img/fig_a1-4-11.svg" alt="取出根節點9，與末尾互換" width="90%">

**② 新根節點 5 往下下沉（Heapify）**：  
比較索引 2（值=8）與索引 3（值=7），左子 8 最大且 8 > 5，**交換**索引 1 與索引 2：

<img src="/img/fig_a1-4-12.svg" alt="Heapify：5↔8，8成新根" width="90%">

**③ 繼續下沉**：索引 2（值=5）的左子索引 4（值=6），右子索引 5（值=2）。  
左子 6 > 5，**交換**索引 2 與索引 4：

<img src="/img/fig_a1-4-13.svg" alt="Heapify：5↔6，繼續下沉" width="90%">

索引 4（值=5）的左子索引 8（值=1）：5 > 1，無須交換，**停止**。

**第一回排序完成：** 8 成為新根，9 已排定於末尾（灰色）。

<img src="/img/fig_a1-4-14.svg" alt="第一回排序完成" width="90%">

### 5.2 後續回合彙整

重複「互換根與末尾 → Heapify」，每回合排定一個最大值：

| 回合 | 取出值 | 互換後新根 | Heapify 後堆頂 | 已排序尾端 |
|:---:|:---:|:---:|:---:|:---:|
| 第 1 回 | 9 | 5 | 8 | `[…, 9]` |
| 第 2 回 | 8 | 1 | 7 | `[…, 8, 9]` |
| 第 3 回 | 7 | 4 | 6 | `[…, 7, 8, 9]` |
| 第 4 回 | 6 | 5 | 5 | `[…, 6, 7, 8, 9]` |
| … | … | … | … | … |
| 第 8 回 | 2 | 1 | — | — |

最終陣列由小到大：`[1, 2, 3, 4, 5, 6, 7, 8, 9]`。

## 六、C++ 程式碼

<CppRunner has-stdin>

```cpp:line-numbers
#include <iostream>
using namespace std;

// 對 arr[i] 往下調整，堆的有效範圍是 1..n
void heapify(int arr[], int i, int n) {
    int largest = i;
    int left  = 2 * i;
    int right = 2 * i + 1;

    if (left  <= n && arr[left]  > arr[largest]) largest = left;
    if (right <= n && arr[right] > arr[largest]) largest = right;

    if (largest != i) {
        swap(arr[i], arr[largest]);
        heapify(arr, largest, n);   // 繼續往下調整
    }
}

// 建立 Max-Heap
void buildHeap(int arr[], int n) {
    for (int i = n / 2; i >= 1; i--)
        heapify(arr, i, n);
}

// 累堆排序主函式（索引從 1 開始）
void heapSort(int arr[], int n) {
    buildHeap(arr, n);              // 第一階段：建堆

    for (int i = n; i >= 2; i--) { // 第二階段：排序
        swap(arr[1], arr[i]);       // 根（最大）移到末尾
        heapify(arr, 1, i - 1);    // 重建堆（範圍縮小）
    }
}

int main() {
    int n;
    cin >> n;
    int arr[n + 1];                 // 從 arr[1] 開始使用
    for (int i = 1; i <= n; i++)
        cin >> arr[i];

    heapSort(arr, n);

    for (int i = 1; i <= n; i++)
        cout << arr[i] << " ";
    cout << endl;
    return 0;
}
```

## 七、時間複雜度分析

| 階段 | 時間複雜度 |
|:---|:---:|
| 建立 Max-Heap（`buildHeap`） | O(n) |
| 每次 Heapify（`heapify`）| O(log n) |
| 排序階段（n 次取出 + Heapify）| O(n log n) |
| **總計** | **O(n log n)** |
| 空間複雜度（原地排序）| O(1) |

> 累堆排序在**最壞、平均、最佳**情況下均為 O(n log n)，且不需要額外記憶體，是一個穩定高效的排序演算法。
